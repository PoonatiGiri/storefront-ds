### Customer-facing storefront

Product browse, search, product detail, cart, checkout — the things customers see. The same `ProductCard`, `PriceDisplay`, and `StatusBadge` work whether you're a single-SKU DTC brand or a 10,000-SKU marketplace.

→ See the [Checkout pattern](/patterns/checkout)

### Order management admin

The operator view — list orders, drill in, fulfill, refund. `DataTable`, `OrderCard`, `OrderSummary`, `FilterPanel`, and the canonical 5-status `StatusBadge` cover the entire lifecycle.

→ See the [Order detail pattern](/patterns/order-detail)

`MetricCard` for KPI tiles, `DataTable` for activity rails, `Tabs` to switch between time ranges. Pair with your charting library of choice — Storefront DS handles everything that isn't the chart itself.

→ See the [Dashboard pattern](/patterns/dashboard)

### Refund & returns workflows

`OrderSummary` natively handles negative totals — pass a negative amount and the component renders correctly with the discount color and `+/-` sign. Pair with `LineItem` for selecting which items to refund.

### Multi-tenant white-label commerce

If you're an agency or commerce-as-a-service platform, retheme per tenant by scoping CSS variable overrides:

```css
[data-tenant="acme"]  { --storefront-color-brand-primary-default: #FF6B35; }
[data-tenant="globex"] { --storefront-color-brand-primary-default: #0066CC; }
```

Every component cascades — change one variable, the whole subtree updates. See [Theming](/theming) for the live playground.

### Headless commerce frontends

Storefront DS doesn't care where your data comes from. Plug it into Shopify Storefront API, Commerce.js, Medusa, BigCommerce, Saleor, or your own GraphQL — the components accept data props, nothing more.
