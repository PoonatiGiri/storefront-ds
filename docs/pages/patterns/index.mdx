# Patterns

Real flows assembled from the components. Patterns aren't reusable code — they're examples of how the primitives compose into the views commerce teams actually ship.

Each pattern below is the same code your app would write. Copy it, change the props, and you have a working page.

- [**Checkout flow**](/patterns/checkout) — Stepper-driven cart → shipping → payment → confirm
- [**Order detail**](/patterns/order-detail) — Order header + line items + summary + status timeline
- [**Dashboard**](/patterns/dashboard) — KPI tiles + recent-orders table snippet

The Storybook has additional flows assembled as stories for visual regression testing.
