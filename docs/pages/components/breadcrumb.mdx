# Breadcrumb

Shows the user's location within a hierarchy and lets them jump back up. Use on detail pages where the user drilled down through a list.

**When to use**: detail views inside nested sections, catalog navigation.
**When not to use**: top-level pages with no parent, linear wizards (use `Stepper`).

```tsx
<Breadcrumb
  items={[
    { label: 'Dashboard', href: '/dashboard' },
    { label: 'Orders',    href: '/orders' },
    { label: 'Order #1024' }, // no href = current page
  ]}
/>
```

## Props

| Prop | Type | Default | Description |
|---|---|---|---|
| `items` | `BreadcrumbItem[]` | _required_ | Ordered list, root → current |
| `separator` | `ReactNode` | chevron | Custom separator element |
| `maxItems` | `number` | — | Collapse middle items if length exceeds this |

### `BreadcrumbItem`

| Field | Type | Description |
|---|---|---|
| `label` | `ReactNode` | Visible label |
| `href` | `string` | Link href; omit for the current page |
| `render` | `(label) => ReactNode` | Custom render (e.g. framework router link) |

## Accessibility

- Renders as `<nav aria-label="Breadcrumb">` with an ordered list inside
- Current page (last item, no `href`) gets `aria-current="page"`
- Separators are `aria-hidden="true"` — decorative
- Use `render` to integrate your router so navigation stays client-side
