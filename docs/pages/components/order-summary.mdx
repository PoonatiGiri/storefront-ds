# OrderSummary

The aggregate-totals card. Use OrderSummary at the bottom of carts, checkout flows, order detail pages, and refund previews.

**When to use**: cart / checkout right-rail, order detail bottom card, refund / return previews, invoice totals.
**When not to use**: a single price (use `AmountDisplay` / `PriceDisplay`), KPI tiles (use `MetricCard`).

```tsx
// Standard checkout summary
<OrderSummary
  currency="USD"
  total={cart.total}
  lines={[
    { label: 'Subtotal', amount: cart.subtotal },
    { label: 'Shipping', amount: cart.shipping },
    { label: 'Tax',      amount: cart.tax, muted: true },
    { label: `Promo (${cart.promoCode})`, amount: -cart.discount, discount: true },
  ]}
  footer={<Button className="w-full">Place order</Button>}
/>

// With line items above the totals
<OrderSummary currency="USD" total={total} lines={lines}>
  {items.map((item) => <LineItem key={item.id} {...item} />)}
</OrderSummary>

// Refund preview — totals can be negative
<OrderSummary
  title="Refund preview"
  totalLabel="Refund total"
  currency="USD"
  total={-89}
  lines={[
    { label: 'Items refunded', amount: -79 },
    { label: 'Tax refunded',   amount: -10, muted: true },
  ]}
/>
```

## Line conventions

| Property | When to use |
|---|---|
| `discount: true` | Promos, store credits — renders in discount red and forces `+/-` |
| `muted: true` | "Estimated"-style lines (e.g. tax before address entry) |
| `hint` | Disclaimer or context beneath the label |
| `amount: null` | When the value can't be computed yet — renders the em-dash placeholder |

## Props

| Prop | Type | Default | Description |
|---|---|---|---|
| `lines` | `OrderSummaryLine[]` | _required_ | Body lines (subtotal, shipping, etc.) |
| `total` | `number \| null \| undefined` | _required_ | Grand total |
| `totalLabel` | `ReactNode` | `'Total'` | Label for the total row |
| `currency` | `string` | _required_ | ISO 4217 currency code |
| `locale` | `string` | `'en-US'` | Locale |
| `title` | `ReactNode` | `'Order summary'` | Card title (set `null` to hide) |
| `children` | `ReactNode` | — | Optional section above the body (e.g. line items) |
| `footer` | `ReactNode` | — | Footer slot (CTA + fine print) |

## Accessibility

- Renders as `<section aria-label="Order summary">`
- Body lines use a semantic `<dl><dt>/<dd>` description list — matches how summaries are typically navigated
- Total is the last `<dt>/<dd>` pair, visually larger
