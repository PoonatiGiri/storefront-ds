# EmptyState

The placeholder shown when a list, table, or section has no content. A good empty state explains *why* and gives the user a clear next step.

**When to use**: first-run experiences before data exists, filtered lists with no matches, disconnected integrations.
**When not to use**: while loading (use `Skeleton`), recoverable errors (use `Alert`).

## Props

| Prop | Type | Default | Description |
|---|---|---|---|
| `title` | `ReactNode` | _required_ | Short, specific title |
| `description` | `ReactNode` | — | Supporting description |
| `icon` | `ReactNode` | _default_ | Custom icon or illustration |
| `primaryAction` | `ReactNode` | — | Primary CTA (e.g. `<Button>`) |
| `secondaryAction` | `ReactNode` | — | Secondary CTA |
| `size` | `'sm' \| 'md' \| 'lg'` | `'md'` | Visual scale |

## Do's and Don'ts

**Do**
- Always provide a meaningful title — never leave the space blank
- Pair the title with a concrete next action when one exists
- Match the empty state's size to the container

**Don't**
- Use empty states to communicate errors — `Alert` is for that
- Stack multiple CTAs of equal weight — one primary, one secondary at most
- Show illustrations that obscure the message
