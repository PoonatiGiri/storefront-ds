# Tabs

Organizes content into parallel sections that share the same screen real estate.

**When to use**: 2–7 sibling views of the same entity (Orders / Customers / Products), filters that don't change the URL.
**When not to use**: one section depends on another, more than 7 sections, multi-step flow (use `Stepper`).

## Components

| Component | Description |
|---|---|
| `Tabs` | Root provider — owns the active value |
| `TabList` | Container for `Tab` triggers |
| `Tab` | Individual trigger; takes a `value` matching a `TabPanel` |
| `TabPanel` | Content shown when its `value` is active |

## Props (Tabs)

| Prop | Type | Default | Description |
|---|---|---|---|
| `value` | `string` | — | Controlled active value |
| `defaultValue` | `string` | — | Uncontrolled default |
| `onChange` | `(value: string) => void` | — | Fires when active changes |
| `variant` | `'line' \| 'pill'` | `'line'` | Visual variant |
| `size` | `'sm' \| 'md'` | `'md'` | Tab size |

## Accessibility

- Implements the WAI-ARIA Tabs pattern: `role="tablist"`, `role="tab"`, `role="tabpanel"`
- Each tab links to its panel via `aria-controls` + `aria-labelledby`
- Arrow keys move focus between tabs, `Home` / `End` jump to first/last
- Only the active tab has `tabIndex=0`; others use `-1` so Tab moves into the panel
- Inactive panels are unmounted to avoid duplicate content for screen readers
