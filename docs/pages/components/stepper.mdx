# Stepper

Communicates the user's position in a multi-step flow. Use for checkouts, onboarding wizards, and any process where progress is meaningful and order is fixed.

**When to use**: checkout (cart → shipping → payment → confirm), multi-step forms, onboarding.
**When not to use**: parallel sections (use `Tabs`), loose progress (use `ProgressBar`).

```tsx
const steps = [
  { label: 'Cart' },
  { label: 'Shipping', description: 'Address & method' },
  { label: 'Payment' },
  { label: 'Confirm' },
];

// Static (read-only)
<Stepper steps={steps} currentStep={1} />

// Allow jumping back to completed steps
<Stepper steps={steps} currentStep={step} onStepClick={setStep} />

// Mark all complete
<Stepper steps={steps} currentStep={steps.length} />
```

## Props

| Prop | Type | Default | Description |
|---|---|---|---|
| `steps` | `StepperStep[]` | _required_ | Ordered list of steps |
| `currentStep` | `number` | _required_ | 0-based index; before = completed, after = upcoming |
| `orientation` | `'horizontal' \| 'vertical'` | `'horizontal'` | Layout direction |
| `onStepClick` | `(index: number) => void` | — | When provided, completed steps become clickable |

### `StepperStep`

| Field | Type | Description |
|---|---|---|
| `label` | `ReactNode` | Display label |
| `description` | `ReactNode` | Supporting description |
| `icon` | `ReactNode` | Custom icon (overrides number / checkmark) |

## Accessibility

- Renders as an ordered list `<ol aria-label="Progress">`
- Current step has `aria-current="step"`
- Clickable completed steps are real `<button>` elements with visible focus ring
- Number/checkmark indicators are visual — labels are the source of truth for screen readers
