# AmountDisplay

The canonical way to render a currency amount anywhere in the system. Always reach for `AmountDisplay` â€” never format currency inline. Handles locale rules, currency symbols, missing values, and strikethrough.

> CLAUDE.md mandates this for every monetary value. Do not `${amount}` anywhere.

## Default

<Preview>
  <AmountDisplay amount={1249.99} currency="USD" />
</Preview>

```tsx
<AmountDisplay amount={1249.99} currency="USD" locale="en-US" />

// Original price (paired with sale)
<AmountDisplay amount={1999} currency="USD" strikethrough muted />

// Credit / adjustment (forces +/-)
<AmountDisplay amount={-49} currency="USD" showSign />

// Missing data â€” renders em-dash
<AmountDisplay amount={null} currency="USD" />
```

## Props

| Prop | Type | Default | Description |
|---|---|---|---|
| `amount` | `number \| null \| undefined` | _required_ | Amount in major units (12.5 = $12.50) |
| `currency` | `string` | _required_ | ISO 4217 currency code |
| `locale` | `string` | `'en-US'` | BCP-47 locale |
| `size` | `'sm' \| 'md' \| 'lg' \| 'xl'` | `'md'` | Visual size |
| `strikethrough` | `boolean` | `false` | Strike through (useful for original price) |
| `showSign` | `boolean` | `false` | Force + sign for positives |
| `muted` | `boolean` | `false` | Muted text color |
| `placeholder` | `string` | `'â€”'` | Rendered when amount is null/undefined |

## Do's and Don'ts

**Do**
- Always pass `amount` in major units (12.5 = $12.50)
- Always pass `currency` â€” it controls symbol, decimal precision, locale-specific formatting
- Use `strikethrough` + `muted` for "was X" original prices

**Don't**
- Concatenate symbols by hand: `${amount}` â€” wrong for INR, JPY, EUR, GBP
- Hardcode `$` anywhere â€” the formatter handles it per currency
- Show `0` when the value is missing â€” pass `null` for the placeholder
