# DataTable

The full table experience for orders, customers, payments, refunds — search, status filtering, pagination, row selection, row actions, plus the three required data-fetching states (loading skeleton, empty, default).

## Variants

DataTable handles four display states via the `variant` prop:

- `default` — normal table with data
- `loading-skeleton` — animated skeleton rows
- `empty-state` — meaningful empty message + action
- `row-selected` / `row-hover` — selection-aware variants

## Props

| Prop | Type | Default | Description |
|---|---|---|---|
| `variant` | `DataTableVariant` | `'default'` | Display state |
| `title` | `string` | — | Table title |
| `rows` | `DataTableRow[]` | — | Data rows |
| `totalCount` | `number` | — | Total record count (footer label) |
| `currentPage` | `number` | `1` | 1-based current page |
| `totalPages` | `number` | `1` | Total pages |
| `selectedIds` | `string[]` | — | Controlled selection |
| `onSelectionChange` | `(ids: string[]) => void` | — | Selection change |
| `onPageChange` | `(page: number) => void` | — | Pagination change |
| `onRowView` | `(row) => void` | — | Row "View" action |
| `onExport` | `() => void` | — | Header Export action |
| `onNewOrder` | `() => void` | — | Header New Order action |

## Composition

Internally composes `SearchBar`, `Pagination`, `Avatar`, `Badge`, and `Button`. Status rendering uses the same five mandated values as `StatusBadge`.
