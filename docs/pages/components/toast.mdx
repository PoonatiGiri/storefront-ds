# Toast

A transient notification that appears, communicates an outcome, and dismisses itself. Use Toast for confirmations and lightweight failures ‚Äî never for blocking decisions.

**When to use**: confirming a save/create/delete, async background results, brief warnings.
**When not to use**: decisions (use `Modal`), persistent status (use `Alert`), per-field validation (use `FormField` error).

## Default

<Preview>
  <Toast variant="success" title="Order saved" onDismiss={() => {}}>
    Your draft order has been saved.
  </Toast>
</Preview>

```tsx
import { ToastProvider, useToast } from 'storefront-ds';

// Wrap your app once
<ToastProvider placement="top-right" defaultDuration={4500}>
  <App />
</ToastProvider>

// Anywhere inside the tree
function SaveButton() {
  const { toast } = useToast();
  return (
    <Button onClick={() => toast({ variant: 'success', title: 'Saved', children: 'Order saved.' })}>
      Save
    </Button>
  );
}

// Persistent toast (must be dismissed manually)
toast({ variant: 'error', title: 'Sync failed', duration: 0, action: <Button size="sm">Retry</Button> });
```

## Placements

The `ToastProvider` accepts a `placement` prop:

`top-right` (default) ¬∑ `top-center` ¬∑ `top-left` ¬∑ `bottom-right` ¬∑ `bottom-center` ¬∑ `bottom-left`

## Props (Toast)

| Prop | Type | Default | Description |
|---|---|---|---|
| `variant` | `'info' \| 'success' \| 'warning' \| 'error'` | `'info'` | Semantic variant |
| `title` | `ReactNode` | ‚Äî | Title (bolded) |
| `children` | `ReactNode` | ‚Äî | Body text |
| `action` | `ReactNode` | ‚Äî | Inline action next to close button |
| `onDismiss` | `() => void` | ‚Äî | Close handler ‚Äî called on X click or auto-dismiss |

## Provider hook

```tsx
const { toast, dismiss } = useToast();

toast({ variant: 'success', title: 'Saved' });        // auto-dismisses after defaultDuration
toast({ variant: 'error',   title: 'Failed', duration: 0 }); // persistent
dismiss(id);                                          // manually dismiss
```

## Accessibility

- The `ToastContainer` carries `aria-live="polite"` ‚Äî screen readers announce new toasts without interrupting
- Each `Toast` is `role="status"`
- Auto-dismiss respects the user ‚Äî keep `defaultDuration` ‚â• 4s
- The dismiss button has an accessible `dismissLabel`
