# Checkbox

Lets users select one or many items from a set, or toggle a single option on/off. Supports an `indeterminate` state for "some-but-not-all" patterns.

**When to use**: multi-select lists, filter panels, "select all" rows, opt-in toggles in forms.
**When not to use**: exclusive choice (use `Radio`), immediate-effect binary setting (use `Switch`).

## Props

| Prop | Type | Default | Description |
|---|---|---|---|
| `size` | `'sm' \| 'md'` | `'md'` | Size |
| `label` | `ReactNode` | — | Label rendered next to the checkbox |
| `helperText` | `ReactNode` | — | Helper text below the label |
| `error` | `boolean` | `false` | Error state |
| `indeterminate` | `boolean` | `false` | Indeterminate state |
| `disabled` | `boolean` | `false` | Native disabled |
| _All other_ | `InputHTMLAttributes<HTMLInputElement>` | — | Standard checkbox attributes |

## Accessibility

- Uses native `<input type="checkbox">` — full keyboard support out of the box (`Space` toggles, `Tab` to move)
- `label` is wired via `htmlFor`/`id`; clicking the label toggles the checkbox
- `aria-invalid` is set when `error` is true and `aria-describedby` links to helper text
- Indeterminate is set imperatively via DOM (matches the native API)
