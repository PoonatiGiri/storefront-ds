# Radio

Lets users pick exactly one option from a small set. Almost always rendered inside a **RadioGroup**.

**When to use**: 2‚Äì6 mutually exclusive options where all should be visible at once.
**When not to use**: more than 6 options (use `Dropdown`), multi-select (use `Checkbox`), immediate toggle (use `Switch`).

## RadioGroup props

| Prop | Type | Default | Description |
|---|---|---|---|
| `name` | `string` | _required_ | Shared form field name |
| `value` | `string` | ‚Äî | Controlled value |
| `defaultValue` | `string` | ‚Äî | Uncontrolled default |
| `onChange` | `(value: string) => void` | ‚Äî | Fires when selection changes |
| `size` | `'sm' \| 'md'` | `'md'` | Applied to every child Radio |
| `disabled` | `boolean` | `false` | Applied to every child Radio |
| `error` | `boolean` | `false` | Applied to every child Radio |
| `orientation` | `'vertical' \| 'horizontal'` | `'vertical'` | Layout direction |
| `label` | `ReactNode` | ‚Äî | Optional group label |

## Accessibility

- `RadioGroup` renders as `role="radiogroup"` with optional `aria-label`
- Arrow keys move between radios in the same group natively
- All children inherit `size`, `disabled`, and `error` from the group ‚Äî override per-radio when needed
