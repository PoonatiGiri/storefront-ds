# Tooltip

A short text label that appears on hover or focus to clarify a control's purpose. Use Tooltips sparingly — if the UI needs constant explanation, redesign the UI.

**When to use**: icon-only buttons, truncated text, disabled controls where the reason isn't obvious.
**When not to use**: critical instructions, touch-only surfaces, interactive content (use `Popover`).

## Default

<Preview padded>
  <Tooltip content="Save changes">
    <Button variant="secondary">Hover me</Button>
  </Tooltip>

```tsx
<Tooltip content="Save changes">
  <Button iconLeft={<SaveIcon />} aria-label="Save" />
</Tooltip>

// Controlled (e.g. onboarding tour)
<Tooltip content="Click to begin" open={showHint}>
  <Button>Start</Button>
</Tooltip>
```

## Props

| Prop | Type | Default | Description |
|---|---|---|---|
| `content` | `ReactNode` | _required_ | Tooltip body |
| `placement` | `'top' \| 'bottom' \| 'left' \| 'right'` | `'top'` | Position relative to trigger |
| `size` | `'sm' \| 'md'` | `'sm'` | Size |
| `open` | `boolean` | — | Force open (controlled) |
| `disabled` | `boolean` | `false` | Disable the tooltip entirely |
| `children` | `ReactElement` | _required_ | Trigger element |

## Accessibility

- Tooltip content is linked to the trigger via `aria-describedby` when open
- Renders with `role="tooltip"`
- Opens on **keyboard focus** as well as hover — never make tooltips mouse-only
- Always provide an `aria-label` on icon-only triggers
