# Design Tokens

All visual values in Storefront DS are expressed as CSS custom properties following the naming convention:

```
--storefront-{category}-{name}-{variant}
```

### Token Reference

| Token | Value | Usage |
|---|---|---|
| `--storefront-color-brand-primary-default` | `#2563EB` | Primary actions, links, focus rings |
| `--storefront-color-surface-background` | `#FFFFFF` | Card and panel backgrounds |
| `--storefront-color-surface-foreground` | `#0F172A` | Body text |
| `--storefront-color-surface-border` | `#E2E8F0` | Dividers, input borders, card borders |
| `--storefront-color-neutral-400` | `#94A3B8` | Placeholder text, secondary labels |
| `--storefront-color-semantic-error-default` | `#DC2626` | Error states, destructive actions |
| `--storefront-color-semantic-success-default` | `#16A34A` | Success states, positive trends |
| `--storefront-color-semantic-warning-default` | `#D97706` | Warning states, pending status |

## Spacing

All spacing tokens are multiples of 4px with sub-scale additions for precision:

<div style={{ display: 'flex', flexDirection: 'column', gap: '6px', marginBottom: '24px' }}>
  {[
    ['0', '0px'],
    ['0-5', '2px'],
    ['1', '4px'],
    ['1-5', '6px'],
    ['2', '8px'],
    ['3', '12px'],
    ['3-5', '14px'],
    ['4', '16px'],
    ['5', '20px'],
    ['6', '24px'],
    ['8', '32px'],
    ['10', '40px'],
    ['12', '48px'],
  ].map(([step, value]) => (
    <div key={step} style={{ display: 'flex', alignItems: 'center', gap: '12px' }}>
      <div style={{ width: '180px', fontSize: '12px', fontFamily: 'monospace', color: '#475569' }}>
        --storefront-spacing-{step}

      <div style={{ height: '12px', background: '#2563EB', borderRadius: '2px', width: value === '0px' ? '1px' : value }} />
    </div>
  ))}
</div>

## Typography

### Scale

| Token prefix | Size | Weight | Line height | Usage |
|---|---|---|---|---|
| `display-lg` | 60px | 700 | 66px | Hero headlines |
| `display-md` | 48px | 700 | 53px | Page titles |
| `display-sm` | 36px | 700 | 41px | Section headers |
| `heading-h1` | 32px | 700 | 40px | Card titles |
| `heading-h2` | 28px | 600 | 36px | Sub-sections |
| `heading-h3` | 24px | 600 | 32px | Panel headers |
| `heading-h4` | 20px | 600 | 28px | List titles |
| `heading-h5` | 18px | 500 | 26px | Component labels |
| `body-lg` | 18px | 400 | 28px | Long-form text |
| `body-md` | 16px | 400 | 24px | Default body |
| `body-sm` | 14px | 400 | 20px | Secondary text |
| `label-md` | 14px | 500 | 18px | Input labels, table cells |
| `label-sm` | 12px | 500 | 16px | Badges, captions |

### Tailwind classes

Typography tokens are consumed via Tailwind utility classes generated from `@theme`:

```tsx
<h1 className="text-h1 font-bold">Page Title</h1>
<p className="text-body-sm text-[var(--storefront-color-neutral-400)]">Caption</p>
<span className="text-label-sm font-semibold">Badge label</span>
```

## Dimension Tokens

Component-specific size tokens that replace hardcoded pixel values:

| Token | Value | Used in |
|---|---|---|
| `--storefront-width-search-bar` | `280px` | TopBar, DataTable |
| `--storefront-width-card-order` | `380px` | OrderCard |
| `--storefront-width-metric-sm/md/lg` | `200/260/320px` | MetricCard |
| `--storefront-height-table-row` | `52px` | DataTable |
| `--storefront-size-indicator-dot` | `6px` | Input, Badge |
