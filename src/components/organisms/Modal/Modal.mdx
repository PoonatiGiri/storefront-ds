# Modal

A focused, blocking dialog rendered above the page. Use Modal when the user must make a decision, confirm a destructive action, or complete a short focused task before continuing.

---

## Overview

**When to use:**
- Confirm destructive actions (delete order, revoke access)
- Short focused forms (create / edit a single record)
- Important notices the user must acknowledge

**When not to use:**
- Long workflows — use a dedicated page or `Drawer`
- Transient confirmation — use `Toast`
- Non-blocking notices — use `Alert`

## Accessibility

- Renders as `role="dialog"` with `aria-modal="true"`
- Title is wired via `aria-labelledby`, description via `aria-describedby`
- Focus is trapped inside the modal — Tab and Shift+Tab cycle within
- Escape closes the modal (disable with `disableEscClose`)
- Overlay click closes the modal (disable with `disableOverlayClose`)
- On open, focus moves to the first focusable element; on close, focus is restored to the trigger
- Body scroll is locked while the modal is open
- Rendered into `document.body` via React portal — so z-index issues with parent stacking contexts are avoided
