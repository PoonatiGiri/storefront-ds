# FilterPanel

A checkbox-based filter panel with grouped options, Apply, and Reset controls. Use FilterPanel alongside DataTable to let users narrow down a data set by multiple criteria simultaneously.

---

## Overview

**When to use:**
- Filtering a data table or list by multiple dimensions (status, date range, value range)
- Sidebar filter panel for a product catalog or order list
- Modal filter sheet on mobile-style layouts

**When not to use:**
- Single-dimension filtering ‚Äî use a Dropdown or a tab bar
- Search-as-you-type filtering ‚Äî use SearchBar

## Usage Examples

```tsx
import { FilterPanel } from './src/components';

const [selected, setSelected] = useState({});

// Sidebar panel (narrow, no close button)
<FilterPanel
  variant="sidebar-panel"
  groups={[
    {
      key: 'status',
      label: 'Status',
      options: [
        { value: 'pending', label: 'Pending' },
        { value: 'completed', label: 'Completed' },
      ],
    },
    {
      key: 'dateRange',
      label: 'Date Range',
      options: [
        { value: 'today', label: 'Today' },
        { value: 'last30', label: 'Last 30 days' },
      ],
    },
  ]}
  selected={selected}
  onChange={setSelected}
  onApply={(sel) => applyFilters(sel)}
  onReset={() => setSelected({})}
/>

// Modal panel (wider, with close button)
<FilterPanel
  variant="modal-panel"
  groups={groups}
  onClose={() => setShowFilter(false)}
  onApply={(sel) => applyFilters(sel)}
/>
```

## Filter groups schema

Each `FilterGroup` has:
- `key` ‚Äî unique string used as the key in the `selected` object
- `label` ‚Äî section heading shown above the options
- `options` ‚Äî array of `{ value: string; label: string }` pairs

The `selected` prop is `Record<string, string[]>` ‚Äî keys match group keys, values are arrays of selected option values.

---

## Do's and Don'ts

**Do**
- Use `onApply` for server-side filtering ‚Äî it fires only when the user explicitly applies
- Use `onChange` for instant client-side filtering ‚Äî it fires on every checkbox change
- Always provide `onReset` ‚Äî users need a way to clear all filters

**Don't**
- Mix `selected` (controlled) and no `selected` (uncontrolled) across renders ‚Äî pick one
- Use more than 4‚Äì5 groups ‚Äî the panel becomes too long to scan

---

## Accessibility

- Checkboxes use native `<input type="checkbox">` with visible labels via `<label htmlFor>`
- Custom checkbox visual is purely decorative ‚Äî the real input handles all a11y
- The close button (modal variant) has `aria-label="Close filters"`
- Apply and Reset buttons have visible text labels
- All interactive elements are keyboard accessible
